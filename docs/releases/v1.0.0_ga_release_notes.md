# SeekerMail v1.0.0 — GA Release Notes (draft)

> Engineering draft for the GitHub Release description (T118). The PM polishes the
> wording before publishing. Windows ships as **Public Beta** at v1.0.

SeekerMail v1.0 is our **first public release** and **first commercial milestone**.
It is a local-first desktop email client where each account is an AI "digital
employee", and your mail never leaves your device except to the AI provider you
configure yourself.

## What's new in v1.0

### Find anything — including inside attachments

SeekerMail now reads the text inside your attachments (PDF, Word, Excel,
PowerPoint, and plain-text files) and makes it searchable. When a search matches an
attachment, the result shows an "in attachment: file.pdf" card; clicking it opens
the email and highlights that attachment.

### One search across every account

Search now spans all of your accounts at once. A new account filter lets you narrow
to one or several accounts, and results are merged and ranked together. Both the
exact-keyword and meaning-based (semantic) searches work across accounts, and the
ranking is stable so the same search gives the same top results every time.

### Deals — a read-only transaction view

Group related emails from different accounts under a single "deal" and see them on
one cross-account timeline, each item colour-marked by its source account. The deal
view is strictly read-only: to reply, you jump back to the original thread. Deals
are your private organising labels — they never become AI memory and never let an
agent read across accounts.

### Windows (Public Beta)

SeekerMail now runs on Windows, with passwords and tokens stored in the Windows
Credential Manager (never in the database), a signed installer, and automatic
WebView2 setup. Windows is a Public Beta at v1.0 while we gather real-world
stability data; macOS is the commercial GA platform.

## Privacy, unchanged

- Your mail stays on your device. The only call SeekerMail itself makes is an update
  check that sends just your version and platform.
- AI requests go only to the provider you set up — there is no SeekerMail server in
  the path.
- No analytics, no telemetry. Logs never contain message bodies, addresses, or keys.
